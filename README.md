# HockeyLine



Локальное Flutter-приложение для управления хоккейной командой: состав, звенья, заметки, статистика, экспорт отчётов и администрирование пользователей.



## Дизайн и UX



- **Тема:** тёмный минималистичный режим (`ThemeMode.dark`). Фон экрана `#121212`, вторичные поверхности `#1A1A1A`, отдельный токен `AppColors.surfaceSecondary` для «второго яруса» карточек (звенья, заметки, админ-список).

- **Акцент:** `#D32F2F` (кнопки, выбранные пункты навигации, столбики графиков, нападающие на круговой диаграмме).

- **Графики:** горизонтальная сетка столбикового графика использует холодный нейтральный цвет `AppColors.chartCool`, чтобы сетка не конкурировала с данными.

- **Нижняя навигация (телефон):** `NavigationBar` с фоном `#2C2C2C` — светлее основного фона, без затемняющего surface tint, чтобы панель читалась отдельно от контента.

- **Карточки игроков:** белые `#FFFFFF`, лёгкая тень, тёмный текст для контраста (`PlayerCardSurface`). Вторичные блоки — `SecondaryCardSurface` в `lib/widgets/design_widgets.dart`.

- **Иерархия и сетка:** отступы и радиусы вынесены в `AppSpacing`, `AppRadius`; длительности анимаций — `AppMotion` (например, переключение вкладок). Типографика — `AppTextStyles` (заголовки экрана/секции, акцентный body, мелкий приглушённый текст, табличные цифры для статистики).

- **AppBar:** опциональная линия «лёд» снизу — `hockeyIceAppBarBottom()` из `design_widgets.dart`.

- **Гость:** под заголовком показывается `GuestModeBanner` с пояснением режима только чтения.

- **Пустые состояния:** единый шаблон `EmptyStateView` (иконка, заголовок, подзаголовок, опциональная кнопка-действие).

- **Обратная связь:** `showAppSnackBar` — обычные, успешные (`success: true`) и ошибочные (`error: true`) всплывающие сообщения с фонами из `AppColors.snackSuccessBg` / `snackErrorBg`.

- **Касания:** целевые области ключевых кнопок (добавить игрока, избранное и т.д.) не меньше **48×48** логических пикселей.

- **Состав:** на карточке игрока номер на майке вынесен в бейдж на аватаре/лидинге; форма игрока разбита на сворачиваемые блоки (`ExpansionTile`), на широком макете номер и амплуа в одной строке.

- **Шрифт:** Roboto (без засечек), удобное чтение на мобильных.

- **Адаптивность:** широкий экран (≥880px) — `NavigationRail` с расширенными подписями при ≥1200px; сетка состава 2–3 колонки от ширины; графики статистики переходят в колонку при узком экране (<520px).



Тема собрана в `lib/theme/app_theme.dart` (`buildHockeyDarkTheme()`, палитра `AppColors`, `PlayerCardSurface`, токены отступов/радиусов/текста).



Переиспользуемые UI-паттерны — в `lib/widgets/design_widgets.dart`.



Светлая тема с фоном `#F5F5F5` в спецификации заложена как возможное расширение; в текущей версии по умолчанию используется только тёмная.



## Что умеет приложение



- Управление игроками: создание, редактирование, удаление, поиск, фильтры, сортировка.

- Работа с заявкой на матч (`inRoster`) и избранными игроками.

- Формирование звеньев drag-and-drop с бизнес-ограничениями по амплуа и вместимости.

- Ведение заметок по игрокам с ролевыми ограничениями.

- Просмотр статистики, быстрые инкременты показателей, графики.

- Экспорт таблицы статистики в `CSV` и `PDF` (PDF с кириллицей через встроенный шрифт Noto Sans); полный бэкап приложения в JSON — в профиле у администратора.

- Импорт полного бэкапа из JSON (тот же формат, что у основного файла и экспорт полных данных в профиле у администратора).

- Локальная авторизация и управление пользователями (`admin`, `coach`, `guest`); после входа сессия сохраняется до явного выхода из аккаунта.



## Роли и права



- `guest` — просмотр интерфейса без изменений данных.

- `coach` — работа с составом, заявкой, избранным, заметками и просмотром статистики.

- `admin` — все возможности `coach` + управление пользователями, полный CRUD игроков и расширенное редактирование статистики.



## Данные и хранение



Приложение работает офлайн и хранит данные в `hockeyline_data.json` (как asset и рабочая копия на устройстве).



- **Android / iOS:** файл в каталоге документов приложения; при первом запуске — копия из bundled `hockeyline_data.json`.

- **Десктоп:** по возможности файл в корне проекта, иначе каталог документов.

- **Web:** `SharedPreferences` (аналог JSON в браузере).



### Бэкап: экспорт и импорт JSON



Экспорт полного JSON-бэкапа доступен в профиле роли **admin** («Экспорт полных данных (JSON)»): в файл попадает то же содержимое, что в рабочем `hockeyline_data.json` (пользователи, игроки, звенья, заметки, `meta`) — удобно для пары «экспорт → импорт». На экране статистики остаются экспорты **CSV** и **PDF** по таблице показателей.



**Импорт:** в профиле (для ролей `coach` и `admin`, не в режиме гостя) — кнопка «Импорт из JSON (бэкап)». После подтверждения выбирается файл `.json`; содержимое **полностью заменяет** локальные данные приложения. Файл должен быть валидным JSON-объектом с полями `users`, `players`, `lines`, `notes` (как при экспорте / в `hockeyline_data.json`). После импорта снова выполняется нормализация и гарантия системной учётки администратора; если текущий пользователь отсутствует в импортированном списке `users`, приложение предложит войти заново.



Реализация: `StorageService.importFullBackupJson`, вызов из `TeamProvider.importBackupJson`, UI в `home_shell_screen.dart` (профиль).



### Сессия после перезапуска



После успешного входа по email/паролю, регистрации или выбора «Продолжить как гость» идентификатор сессии сохраняется в `SharedPreferences`. При следующем запуске приложения (`AuthProvider.bootstrapInitialSession` из `lib/app.dart`) выполняется восстановление пользователя из файла данных по сохранённому id (для гостя — снова гостевая сессия). **Повторный ввод пароля нужен только после «Выйти из аккаунта»**, удаления аккаунта или если пользователь пропал из данных (например, после импорта другого файла).



### Автонормализация JSON



При чтении и сохранении выполняется нормализация структуры: ключи `users`, `players`, `lines`, `notes`, поле `meta.updatedAt`.



### Формат `hockeyline_data.json`



```json

{

  "users": [],

  "players": [],

  "lines": [],

  "notes": [],

  "meta": {

    "updatedAt": "2026-04-17T12:00:00.000Z"

  }

}

```



## Звенья: правила и ограничения



Типы линий: `goalkeeper`, `forward`, `defense`. Ограничения по амплуа и вместимости (2 / 3 / 2 игрока) — см. код `LinesProvider`. При нарушении показывается сообщение через `showAppSnackBar` (ошибка с красноватым фоном).



Обязательные id линий при миграции: `goalkeeper-1..2`, `forward-1..4`, `defense-1..3`.



## Игроки и фото



Поддерживаются URL и локальные пути; отображение в списке, карточке и форме.



## Статистика и графики



Круговая диаграмма по амплуа, топ-5 по очкам, список с быстрыми действиями для `admin`.



## Экспорт и импорт



- **Экспорт:** полный JSON-бэкап и импорт — из профиля (см. раздел «Бэкап»); `CSV`/`PDF` со страницы статистики — по возможности в публичную папку **Download** на Android, иначе в каталог документов приложения.

- **Импорт JSON:** только из профиля, см. раздел «Бэкап: экспорт и импорт JSON» выше.



## Валидации



Email, пароль, имя/фамилия, дата рождения, номер, рост, вес, фото — см. `lib/utils/validators.dart`.



## Архитектура



- `models`, `services`, `providers`, `screens`, `theme`, `utils`.

- `lib/theme/app_theme.dart` — тема, цвета и дизайн-токены.

- `lib/screens/home_shell_screen.dart`
